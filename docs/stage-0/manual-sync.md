# Ручная синхронизация событий (Этап 0)

Этап 0 предполагает ручное управление выгрузкой данных из локального логгера в общее GitHub-хранилище.

## Подготовка

1. Убедитесь, что локальное окружение использует Python 3.10+.
2. Установите пакет проекта в editable-режиме (инструкции появятся после публикации `pyproject.toml`). Для прототипа можно запускать скрипты напрямую.
3. Создайте личную копию репозитория `memory-store` (приватный GitHub-репозиторий для хранения событий).

## Запись событий

```bash
python scripts/demo_logger.py
```

Скрипт продемонстрирует создание событий и сформирует файл экспорта в каталоге `outbox/`.

## Проверка и очистка

- Перед отправкой просмотрите JSON-файл и убедитесь, что он не содержит секретов, приватных путей (`C:\Users\`, `~/Downloads` и т.д.) или бинарных артефактов.
- При необходимости вручную отредактируйте/замаскируйте конфиденциальные поля.

## Публикация в GitHub memory-store

1. Скопируйте файл из `outbox/events-YYYYMMDDTHHMMSSZ.json` в локальную копию репозитория memory-store (например, в каталог `incoming/`).
2. Добавьте файл в git, создайте коммит и откройте pull request.
3. Дождитесь ручного ревью и мержа.

## Постобработка

- После успешной синхронизации можно удалить файл из `outbox/` или сохранить для архива.
- В дальнейшем эти действия будут автоматизированы CLI-инструментом `memoryctl`.
