# План: Marketplace Packages

> Создано: 2025-11-10  
> Основано на spec-driven workflow (см. `docs/research/spec_kit_analysis.md`).

## Контекст
- Marketplace API реализован (загрузка артефактов, модерация, аудит), но нет готовых пакетов, которые можно показать пользователям и использовать как эталон.
- Первые кандидаты — `onec-markdown-viewer` и `VAEditor` (см. `docs/research/marketplace_integration_plan.md` и репозитории @alkoleft), востребованные среди разработчиков экосистемы.
- Фича нужна продукту (витрина Marketplace), команде DevRel (материалы и демо) и QA (смоук-тесты пакетов).
- Связанные документы: `docs/research/marketplace_integration_plan.md`, `ADR-0003-marketplace-backend.md` (архитектура), `docs/scripts/README.md` (cli).

## Цели
- [ ] Подготовить и опубликовать два эталонных пакета (`onec-markdown-viewer`, `VAEditor`) в Marketplace (priv → prod).
- [ ] Автоматизировать упаковку и загрузку (Make/CLI) с чек-листом артефактов и манифестов.
- [ ] Обновить документацию (README, docs/06-features, FAQ) и сценарии QA.

## Нецели
- Массовая портировка всех репозиториев @alkoleft (только два первых пакета).
- Выкладка платёжной модели (billing API будет оцениваться отдельно).
- Разработка полноценного UI для модерации (используем существующий админ-интерфейс).

## Выходы
- bundle zip для каждого пакета (`plugin/`, `docs/`, `assets/`, checksum).
- `plugin.json` + README/CHANGELOG/FAQ, скриншоты и иконки.
- Скрипт(ы) в `scripts/marketplace/` + make-таргеты `package-<plugin>`, `publish-<plugin>`.
- Гайд `docs/06-features/MARKETPLACE_PACKAGES.md` и обновлённый FAQ.
- Метрики: успешная публикация, смоук-загрузка, наличие в витрине.

## Таймлайн и зависимости
- Iteration 1 (1 неделя): ревизия репозиториев, шаблон манифеста, packaging-скрипты.
- Iteration 2 (1 неделя): автоматизация публикации, документация, QA сценарии.
- Iteration 3 (буфер): маркетинг (скриншоты, видео), проверка откатов.
- Зависимости: доступ к MinIO/S3, учётные записи Marketplace, апрув DevRel/QA.

## Риски и предположения
- Юридические ограничения по лицензиям внешних репозиториев (решение — запрос у авторов, указание лицензий).
- Возможные изменения API Marketplace — держим ADR и doc в актуальном состоянии.
- Ограничения CI на размер артефактов — проверить лимиты GitHub Actions/MinIO.

- [ ] Прогнаны обязательные скрипты и тесты (см. `docs/research/constitution.md`).
- [ ] Благодарности внешним проектам указаны (репозитории @alkoleft, авторы пакетов).
