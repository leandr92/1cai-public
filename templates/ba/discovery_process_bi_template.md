# Discovery → Process → BI Scenario Template (Sprint 1)

## 1. Цель

- Повторяемый сценарий исследования процесса, моделирования и визуализации данных.
- Используется тренажёрами (Discovery Agent, Process Modeler, Storytelling Coach).
- Обеспечивает единый стандарт онбординга BA и проведения воркшопов.

## 2. Подготовка

- **Артефакты**: описание процесса as-is, данные из 1С (выгрузка), список стейкхолдеров.
- **Участники**: Product Owner, BA, Tech Lead, представитель 1С, Data Analyst.
- **Инструменты**: Miro/Whiteboard, 1С, Camunda/Bizagi, Power BI/DataLens, BA pipeline.

## 3. Шаги сценария

### 3.1 Discovery Session
- Цель: собрать требования, pain points, KPI.
- Шаблон вопросов:
  - Расскажите о текущем процессе и его целях.
  - Какие метрики отслеживаете? Где данные считаются?
  - Какие ограничения (регуляторика, ресурсы, интеграции)?
  - Путь пользователя: где основные задержки?
- Документация: заполнить Discovery Canvas (см. раздел 4).

### 3.2 Process Modeling
- Input: выводы discovery, список акторов, data points.
- Output:
  - BPMN as-is (указать bottlenecks).
  - BPMN to-be (целевая автоматизация, swimlanes).
  - Решения decision points (DMN, условия SLA).
- Проверка: Digital BA Lead оценивает completeness, добавляет рекомендации.

### 3.3 BI & Analytics
- Подготовить дашборд SLA: время обработки заявок, backlog, throughput.
- Предложить улучшения: какие метрики добавить, как делать storytelling.
- Выходные артефакты:
  - Power BI/DataLens черновик (JSON/файл).
  - Business narrative (Storytelling Coach) для презентации.

## 5. Checklist (to-be готовность)

- [ ] Требования собраны, приоритеты проставлены (MoSCoW).
- [ ] BPMN as-is/to-be оформлены и согласованы.
- [ ] BI/аналитика подготовлены (dashboards + narrative).
- [ ] Риск-регистр обновлён.
- [ ] План внедрения (roadmap) согласован.

## 6. Связь с BA Pipeline

- После каждой сессии данные загрузить через `scripts/ba_internal/artefact_ingest.py`.
- Обновить assessment формы (`scripts/ba_assessment/generate_forms.py`) — учитываем новые навыки.
- Настроить напоминание: повторять сценарий каждые 3 месяца или при запуске новых проектов.
